# Maneater

Man page search index and semantic search CLI. Uses vector embeddings
(nomic-embed-text-v1.5 via llama.cpp CGo bindings) to build a searchable index
over Unix man pages.

## Build & Test

```sh
just build          # go build -o build/maneater ./cmd/maneater
just test           # go test ./...
just generate       # go generate (regenerate config_tommy.go)
just gomod2nix      # regenerate gomod2nix.toml
just build-nix      # nix build --show-trace
just fmt            # gofumpt + goimports
```

## Architecture

- `cmd/maneater/` -- CLI binary (index + search subcommands)
- `internal/embedding/` -- CGo bindings to llama.cpp for vector embeddings

CGo requires `CGO_ENABLED=1`, `pkg-config`, and `llama-cpp` in buildInputs.

## Configuration

Hierarchical TOML config (`maneater.toml`):
1. `~/.config/maneater/maneater.toml` (global)
2. Parent directories between `$HOME` and `$CWD`
3. `./maneater.toml` (project-local)

Generated code: `config_tommy.go` is generated by `//go:generate tommy generate`.
Do not edit it manually.

## Testing

- Pure Go tests (cosine, index, config, manpath) run without external deps
- Embedding tests require `MANPAGE_MODEL_PATH` env var pointing to a GGUF model
- `search_quality_test.go` documents expected ranking behavior and known gaps

## Nix

Follows the stable-first nixpkgs convention:
- `nixpkgs` = stable (runtimes, core tools)
- `nixpkgs-master` = latest (Go toolchain)

The wrapped binary bundles mandoc, pandoc, tldr on PATH and sets
`MANEATER_CONFIG` to point to bundled model configs.
